Geometric Mean

This will be all the math involved with the Geometric Mean (G3M) trading function.

Conceptual Overview

G3M (for two assets) consists of a pool of reserves and their associated weights.

The G3M effectively gives the LP a portfolio that consists of a fixed ratio of the two assets based on the internal pricing mechanism. For instance, if we pick the weight of the X-token 0.80 and 0.20 for the Y-token, then the LP will have a portfolio that is 80% in X and 20% Y by price. This is a basic building block for a lot of portfolio designs.

Core

Mechanically, G3M of two variable parameters:

  • wx​≡weight_x
  • wy​≡weight_y
  • These parameters must satisfy wx​,wy​≥0wx​+wy​=1

Next, we define the trading function to be: φ(x,y)=xwx​ywy​=L where L is the invariant of the pool. We can put: L≡liquidity Note that L is in units of Token by virtue of the geometric mean.

Price

If we compute the derivatives and simplify the expression, we get that the pool price is: P=wy​wx​​xy​​ We can determine a price in terms of just x or just y if need be.

Swap

We require that the trading function remain invariant when a swap is applied, that is: L(x,y)=(x+Δx​)wx​(y+Δy​)wy​ while also taking fees as a liquidity deposit (which will increase the liquidity L).

Trade in ΔX​ for ΔY​

Suppose that we want to trade in ΔX​ for ΔY​. Then we have that we are really inputting γΔX​ while raising L↦L+δL​. This gives us a new invariant, noting that we can create Noting that y=wx​wy​​px LX​(x,S)=x(wx​wy​​S)wy​​ LX​(x+aδx​)=LX​(x)+aLX​(δX​) which also tells us: x=(wx​wy​​S))wy​L​ and note that LX​(x,p) is linear in x. Then ΔX​(ΔY​)=((y+γΔY​)wy​L+δLY​​​)1/wx​−x​

Liquidity Provision

It must be that adding liquidity does not change the price of the pool. This makes it quite simple to add liquidity. If a user wants to add liquidity, they can just add the tokens such that the ratio of the reserves does not change. If a user wants to input Δx​ and Δy​ to the pool, then they must have: p=wy​wx​​xy​=wy​wx​​x+Δx​y+Δy​​ which implies if they choose a given Δx​, then they must have: Δy​=xy​(x+Δx​)−y and similarly if they choose a given Δy​, then they must have: Δx​=yx​(y+Δy​)−x

Value Function via L and S

Given that we treat Y as the numeraire, we know that the portfolio value of a pool when X is at price S is: V(x,y,S)=xS+y We can find the relationship to portfolio value from V(L,S). This will be helpful when tokenizing pool LP positions.

Since we have LX​(x,S) and LY​(y,S), we can get the following: x=(wx​wy​​S)wy​L​y=L(wy​wx​​S1​)wx​​ Therefore: V(L,S)=(wx​wy​​S)wy​LS​+(wy​wx​​S1​)wx​L​V(L,S)=LSwx​((wy​wx​​)wy​+(wx​wy​​)wx​)​ Note that V is linear in L and so we can use this to tokenize.
